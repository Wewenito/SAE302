Server Module#

class server.Server_handle(host, port)[source]#

Bases : object

Ceci est la classe principale du serveur. Plusieurs threads sont créés basés sur des méthodes de la classe pour chacun des clients se connectant.

Paramètres:

  • host (str) – Adresse IP de la machine hôte où le serveur est hébergé.

  • port (int) – Port utilisé par le serveur (11111)

Variables:

  • server_socket (object) – Socket utilisé par le serveur pour communiquer avec les clients

  • current_threads (array) – Tableau contenant pour chaque client connecté les informations suivantes : [Nom de la connection (4 chiffres), Objet de connection, Utilisateur authentifié ou non].

  • user_data (array) – Tableau contenant pour chaque client connecté les infos suivantes : [Nom de la connection (4 chiffres), dictionnaire contenant les informations de l’utilisateur.]

CHandler(command)[source]#

Cette méthode s’occupe d’analyser un message reçu de la part d’un client et renvoie l’action à effectuer.

Paramètres:

command (str) – Message à analyser.

Cette méthode est toujours appelée par le thread principal d’écoute de chaque client “listen_to_user”.

ban_user(user_to_ban)[source]#

Cette méthode est utilisée pour bannir un utilisateur.

Avertissement

Cette action est définitive et irréversible !

Paramètres:

user_to_ban (str) – Nom de l’utilisateur à bannir.

Variables:

  • mydb (Object) – Objet de connection Mysql

  • user_data (dictionnary) – Données de l’utilisateur

Note

Si l’utilisateur banni est actuellement connecté, on récupère son adresse ip et associe celle-ci au ban. Egalement, on déconnecte de force l’utilisateur.

conn_buffer(connection, exit_flag)[source]#

Cette méthode est utilisée à chaque fois qu’un nouveau client se connecte au serveur. Ceci permet de ne pas bloquer l’accès au serveur à d’autres clients si quelqu’un est d’ores et déjà connecté mais pas encore authentifié.

Le but de cette méthode est d’écouter si le nouveau client détecté souhaite s’enregistrer ou se connecter.

Paramètres:

  • connection (socket object) – Objet de connection utilisé pour communiquer avec le client.

  • exit_flag (Object) – Ce flag est utilisé pour la fonction /kill

Variables:

action (str) – Message reçu de la part du client.

Avertissement

Si le message reçu par le client est vide, cela signifie que celui-ci à été déconnecté. Dans ce cas, on retire le client de la liste des clients actifs.

Renvoie:

Démarre le thread associé à la requête du client.

Type renvoyé:

Action

kick_user(user_to_kick, duration_of_the_kick)[source]#

Cette méthode est utilisée pour kick un utilisateur.

Paramètres:

  • user_to_kick (str) – Nom de l’utilisateur à kick.

  • duration_of_the_kick (int) – Durée du kick de l’utilisateur (en secondes).

Variables:

  • mydb (Object) – Objet de connection Mysql

  • user_data (dictionnary) – Données de l’utilisateur

Conditions

  • Si l’utilisateur n’existe pas -> Arrêt de la méthode et retour “Faux”

  • Si l’utilisateur est déjà banni -> Arrêt de la méthode et retour “Faux”

  • Sinon, l’utilisateur est bien kick.

Note

Cette méthode vérifie également si l’utilisateur à kick est actuellement connecté au serveur. Si tel est le cas, alors on le déconnecte de force.

Renvoie:

False (if error)

Type renvoyé:

bool

listen_for_conns(exit_flag)[source]#

Cette méthode écoute en boucle la connection de nouveaux clients tant que le flag d’arrêt n’est pas actif. Lorsqu’une connection est détéctée, la méthode “conn_buffer” est enclenchée pour ne pas bloquer l’accès à d’autres clients.

Paramètres:

exit_flag (Object) – Ce flag est utilisé pour la fonction /kill

Note

Timeout

Un timeout de 2 secondes est appliqué à la réception du message afin de ne pas bloquer la commande /kill au cas ou celle-ci viendrait à être appelée.

Lève:

  • socket.timeout – Si l’écoute dépasse le timeout de 2s.

  • Exception – Si la méthode rencontre un comportement imprévu.

listen_to_user(connection, exit_flag)[source]#

Cette méthode représente la boucle principale d’écoute de chaque client. Celle-ci fonction de la manière suivante :

  • Reception du message,

  • Analyse du message avec la méthode “CHandler”

  • Action à effectuer suite à l’analyse.

Paramètres:

  • connection (socket object) – Objet de connection utilisé pour communiquer avec le client.

  • exit_flag (Object) – Ce flag est utilisé pour la fonction /kill

Avertissement

Si le message reçu est vide, cela signifie que le client à été déconnecté, on retire alors ses informations associées du serveur.

Action réalisable

  • [BROADCAST] -> Envoie d’un message de broadcast

  • [ROOM_MESSAGE] -> Envoie d’un message sur un salon de discussion

  • [PRIVATE_MESSAGE] -> Envoie d’un message sur une conversation privée

  • [GATHER_ROOM_MESSAGES] -> Récupère tout les messages d’un salon

  • [USER_QUERY_FOR_ROOM_RIGHTS] -> Soumission d’une requête utilisateur pour rejoindre un salon.

  • [REQUEST_FOR_ADMIN_DEMANDS] -> Récupère les demandes d’utilisateurs pour rejoindre des salons.

  • [REQUEST_CHANGE_USER_ROOM_RIGHTS] -> Mise à jour des droits d’accès à un salon pour un utilisateur suite à la décision d’un admin

  • [GATHER_PRIVATE_MESSAGES] -> Récupère tout les messages d’une conversation privée.

  • [FRIEND_REQUEST] -> Ajout d’une demande d’ami de la part d’un utilisateur

  • [ACCEPT_FRIEND_REQUEST] -> Mise à jour d’une relation entre deux utilisateurs suite à la décision de l’un d’entre eux.

Lève:

  • socket.timeout – Si le timeout est dépassé

  • Exception – Si une erreur imprévue est survenue durant la réception.

log_user(client_conn, exit_flag)[source]#

Cette méthode est le processus principal de connexion d’un utilisateur. Celle-ci consiste de multiples échanges entre le serveur et le client.

Paramètres:

  • client_conn (Object) – Objet de connexion avec le client.

  • exit_flag (Object) – Ce flag est utilisé pour la fonction /kill

Variables:

  • response (str) – Réponse reçu du client.

  • mydb (Object) – Objet de connection Mysql

  • user_data_proposed (str) – Association « Nom utilisateur|Mot de passe » reçu du client.

Note

Un timeout est instauré sur la réponse de client afin de vérifier toute les secondes si le serveur doit toujours être actif et l’arrêter si une commande /kill est émise.

Conditions pour une connexion réussie

  • L’utilisateur doit exister.

  • L’utilisateur ne doit pas être banni et/ou kick.

  • Le mot de passe fourni correspond à celui stocké dans la base de données.

Lève:

BrokenPipeError – Si le client est déconnecté durant le processus.

Renvoie:

Si l’utilisateur est bien valide, on récupère ses amis, ses droits d’acès, les salons connus et lui renvoie l’entièreté de ces infos. Un thread d’écoute associé à ce client est ensuite initié.

Type renvoyé:

Action

register_user(client_conn, exit_flag)[source]#

Cette méthode est le processus principal d’inscription d’un utilisateur. Celle-ci consiste de multiples échanges et vérifications entre le serveur et le client.

Paramètres:

  • client_conn (Object) – Objet de connexion avec le client.

  • exit_flag (Object) – Ce flag est utilisé pour la fonction /kill

Variables:

  • response (str) – Réponse reçu du client.

  • mydb (Object) – Objet de connection Mysql

  • banned_IPs (array) – Liste des adresses IP associées à des utilisateurs bannis

  • current_ip (str) – Adresse IP du client souhaitant s’inscrire.

Note

Un timeout est instauré sur la réponse de client afin de vérifier toute les secondes si le serveur doit toujours être actif et l’arrêter si une commande /kill est émise.

Conditions pour une inscription réussie

  • L’adresse IP du client n’est pas une adresse bannie.

  • L’utilisateur doit exister.

Les autres conditions sont effectuées coté client.

Lève:

BrokenPipeError – Si le client est déconnecté durant le processus.

Renvoie:

Une fois le nouvel utilisateur créé, on ajoute également à la base de données ses droits d’accès aux salons. Enfin, on lance un thread d’écoute sur le nouvel utilisateur.

Type renvoyé:

Action
